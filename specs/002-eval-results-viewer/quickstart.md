# Quickstart: Eval Results Viewer

## Prerequisites
- Node.js (v18+)
- Local `tests` directory containing evaluation data

## Setup

1. **Install dependencies**
   ```bash
   npm install
   ```

2. **Run the Viewer**
   The viewer expects a `tests` directory in the project root containing `eval_dataset`, `eval_results`, and `llm_invoke_results`.
   ```bash
   npm run view
   ```
   *This command will execute the local CLI script which parses the evaluation data and starts a local web server (e.g., using Vite) to serve the visualization app. It will then automatically open your browser.*

## Architecture Overview
- **Data Parser**: A local Node.js script (`eval-view.ts`) at the project root that reads and merges local JSON/JSONL files on-demand into static resources.
- **Frontend**: A React application located in `ui/` that consumes the parsed local data and renders the summary list and detailed chat UI.
- **Data Source**: The application is entirely file-backed, relying on the outputs generated by the evaluation framework. No traditional database or heavy backend service is used.
