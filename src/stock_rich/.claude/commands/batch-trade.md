批量期权交易分析：读取 batch.md → 逐个执行 /trade → 生成汇总报告。

## 用法
`/batch-trade`

自动读取 `.claude/batch.md` 中的交易列表，逐个分析。

---

## Phase 1: 解析批量任务

读取 `.claude/batch.md`，提取所有交易任务。

格式要求：
```
股票代码 到期日 方向
```

示例：
```
NVDA 2026-03-07 call
TSM 2026-03-14 put
ASML 2026-02-28 call
```

解析规则：
- 跳过空行和注释行（以 `#` 开头）
- 跳过代码块标记（```）
- 每行必须包含 3 个字段：symbol, expiry (YYYY-MM-DD), direction (call|put)
- 验证日期格式和方向有效性

展示解析结果：
```
📋 批量任务清单：
1. NVDA 2026-03-07 call
2. TSM 2026-03-14 put
3. ASML 2026-02-28 call

共 3 个任务
```

### ⏸️ 暂停点：确认任务列表
> 以上是解析到的交易任务。你可以：
> 1. 编辑 batch.md 调整任务列表
> 2. 直接回复"继续"开始批量分析

---

## Phase 2: 逐个执行分析

对每个任务，按照 `/trade` 的完整流程执行：

### 2.1 数据采集（复用检查）
- 检查 stockdata.json 是否已有该 symbol
- 检查 news-{SYMBOL}.json 是否已存在
- 始终重新获取 options 数据

### 2.2 执行分析
调用 `/trade` 的核心逻辑：
1. 获取基本面+技术面（如需）
2. 获取期权量化数据
3. 获取消息面+内幕交易（如需）
4. LLM 综合研判
5. 生成独立报告 → `output/trade-{SYMBOL}-{EXPIRY}-{direction}.md`

### 2.3 进度显示
```
[1/3] ✅ NVDA 2026-03-07 call - 完成
[2/3] 🔄 TSM 2026-03-14 put - 分析中...
[3/3] ⏳ ASML 2026-02-28 call - 等待中
```

### 2.4 错误处理
如果某个任务失败（API 错误、数据缺失等）：
- 记录错误信息
- 继续执行下一个任务
- 在汇总报告中标记失败项

## 使用建议

### 典型场景
1. **周末复盘**: 分析下周所有关注标的
2. **板块轮动**: 批量分析同板块多只股票
3. **财报季**: 批量分析即将财报的股票

### 效率优化
- 数据复用：stockdata.json 和 news-*.json 自动去重
- 并行可能：未来可考虑并行获取数据（当前串行执行保证稳定性）

### Token 预算控制
- 建议单次批量不超过 10 个任务
